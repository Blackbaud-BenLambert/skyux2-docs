The inline form component renders a form in the current view rather than opening a modal.
Usage
When to use inline forms
  • The add/edit form is simple in terms of a small # of fields and low complexity of input controls
  • The surrounding content will not distract the user from the add/edit task
  • There is a high likelihood the user would need to add multiple records in succession
  • Adding/editing a subsection within a larger modal form
  • Itâ€™s ok if the user abandons the task while the add/edit form is open (i.e. the task is functionally non-modal)
When not to use inline forms
  • Add/edit form would lead to a large amount of page morphing or a user needing to scroll to complete the form if done inline
  • Working with a paginated or infinite scroll list of items
  • You want the user to have to finish the add/edit before doing anything else (i.e. the task is functionally modal)

Behavior
Add form

When the user clicks the Add button, the form should slide out from the bottom of the list.

Edit form

When the user clicks the Edit button, the view content should fade out and the form should slide in to replace the view content.
