Like most design systems, SKY UX follows a roughly atomic-design structure where basic elements are combined and composed into larger constructs.

However, the primary use cases we support lead to an important distinction - SKY UX is much more prescriptive than other design systems you may have used before. The majority of our consumers are building within the Blackbaud ecosystem, and as such we define more rigorous guidelines to help ensure a cohesive experience for our customers.

Principles

High-level considerations that help inform design direction.

Ultimately, all design decisions made within the design system roll up to principles.

Guidelines

Patterns describing how to use and combine components and style elements to build applications.

Components

The user interface elements with which you’ll build your applications.

Styles

The basic elements from which components are constructed. Often times you won’t need to worry about these - the components will handle them for you.
