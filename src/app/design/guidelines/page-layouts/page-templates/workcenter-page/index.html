Usage
The workcenter layout surfaces time-sensitive information and allows users to quickly access the most important tasks and information in the application.
Anatomy
Workcenters are a flexible layout concept, adaptable to the specific role or task being supported. Some common page elements include a page summary to highlight the most important information; a tabbed layout to support different tasks within a role; and tiles to allow the user to arrange the page in the way that best suits their usage.
  1. Page summary
  2. Tabbed views (optional)
  3. Content area
Examples
Simple

A simple fluid grid-based layout is appropriate when:

  • Important information can be presented in a compact format.
  • User does not need to rearrange content.
Multiple tasks

A tile-based layout is appropriate when:

  • The page is supporting multiple tasks which each have important information to show.
  • Certain tasks may be more or less important to an individual user, so being able to rearrange content is valuable.
Multiple responsibilities

A tab-based layout is appropriate when:

  • The page is supporting a role with multiple responsibilities which each have a lot of important information to show.
Interactions
Accessing details

To keep the user in context, use a flyout to show the details behind summary information.
