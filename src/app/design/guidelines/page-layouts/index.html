Page structure
1. Primary navigation

SKY UX places primary navigation at the top of the page.

2. Page title

The page title uniquely identifies the page so the user knows exactly where they are.

3. Page summary (optional)

Place important information in the page summary. If the information will be important to see every time the user views the page, or would cause them to take action upon seeing the information, it's a good candidate for the page summary.

4. Page toolbar (optional)

Place buttons invoking page-level actions in the page toolbar. Actions related to child data should be placed with that data rather than in the page toolbar.

5. Multiple views (optional)

Some pages will have a large amount of content that can be divided among multiple views to support different tasks.

6. Main content

The main content area can take many different forms, depending on the scenario you are designing for.
Page layout templates

SKY UX is a flexible system that allows for a variety of page layouts to solve workflow needs while maintaining a consistent feel across an application.

The following sections describe some common SKY UX page layouts. These layouts show how to package components to suit a variety of user needs, but the system can accommodate alternate layouts as necessary.

Record page

The record page layout provides a tile-based page that displays a page summary and a main content area. The page summary at the top of the page provides an overview of a record. The main content area usually consists of two columns of tiles (or one column on extra small screens) that display information about the record in logical sections. The content to include in each area depends on the page's use case.

List page

The list page layout provides a page that displays a list of related data. Common controls allow users to select the columns to display, sort the list, filter the list, search, and switch views. List pages can use tabs to display multiple lists or pre-filtered instances of a single list.

Workcenter page

The workcenter layout provides a tile-based page that surfaces time-sensitive information and allows users to quickly navigate to the most recent information in an application. It can contain a page summary to highlight the most important information, a tabbed layout to support different tasks, and a carousel to feature high-priority items.

Split view page

Split views are used in scenarios where the user will be performing a task for each of a set of records. The left panel contains the records the user will work through, and the right panel contains the information and actions they need to complete the task for the selected record.

Accessibility
HTML <title>

Wording guidelines:

  • Keep titles short.
  • Make titles unique.
  • Provide concise descriptions of each page's contents or purpose. Titles should stand on their own and make sense out of context.
  • Start with the most meaningful, differentiating terms so that they can be scanned in browser tabs, window lists, and browser bookmarks.

Use the following format for the page's HTML <title> element:

<Page title> - <Product title>

Example: Gifts list - Raiser's Edge NXT

For pages with tabbed subnavigation, use the following format:

<Tab title> - <Page title> - <Product title>

Example: Portfolio - Work center - Raiser's Edge NXT
